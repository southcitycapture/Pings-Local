The whole window is the peer list

No settings panel, no debug readouts, no tabs. One row per person, one action per row.

Hover reveals the two verbs

Ping and Message appear on hover; double-click a row pings. Amber belongs to the ping action alone — it's the one loud color in the app.

State is form, not prose

A presence dot, an unread badge, a "Sent ✓" chip. No mono metadata paragraphs under every card — the IP and last-seen live in one quiet line.

Identity survives DHCP

Rows are people (stable peer IDs), not IP addresses. Aliases, sounds, and history follow the person when their address changes.

Settings live in Settings

One settings window. The gear is the only way in, and nothing in the main window needs saving.

The fast paths

The window is optional. The overlay, the palette, and the DM window are where Pings actually gets used.

What this replaces

  • One accent instead of five gradients. v2 puts a teal→teal gradient on every button; v3 spends color only where it means something.
  • Tokens instead of hard-coded teal. v2's dark mode half-breaks because metadata, tabs, and summaries hard-code light hex values.
  • A buddy list instead of a three-panel dashboard. The settings form, the tab bar, and the activity log all leave the main window.
  • Designed sounds instead of oscillator beeps, bundled with the app like the fonts.
  • Real delivery states instead of fire-and-forget UDP — the ✓✓ in the DM mock is a protocol feature, not decoration.
